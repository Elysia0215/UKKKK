## 3. 프론트엔드 변경 내용

### 3-1. `GapMatrixDashboard.tsx`

핵심 목적:

- Gap Matrix 상세/비교 모달에서 단순 원천데이터 나열이 아니라 "이 데이터로 어떤 정책 가설을 세울 수 있는지"를 바로 보여주도록 개선했다.
- Academic Proof가 논문 소개 카드가 아니라 현재 선택된 문제 클러스터의 판단 근거 레이어로 보이도록 재구성했다.

주요 구현:

- **산점도 차트 및 아코디언 일람표 간의 양방향 반응형 인터랙션 구현**:
  - `handleCardClick` 함수를 개선하여 ScatterChart의 버블 클릭 시, 해당 클러스터가 속한 대분류(`issue.category`)를 아코디언에서 자동으로 펼칩니다 (`expandedCats` 상태 업데이트).
  - 아코디언이 전개된 후 DOM 렌더링 타이밍을 맞추기 위해 `setTimeout` (120ms) 딜레이를 부여해 해당 행(`tr`의 ID `row-${issue.id}`)으로 부드럽게 스크롤 정렬(`scrollIntoView({ behavior: 'smooth', block: 'center' })`)하는 로직을 연동했습니다.
  - ScatterChart Cell 렌더링 시 `selectedIssue`와 매칭되는 버블(`isSelected`)을 불투명도 `1.0`, 진한 Slate 테두리(`#020617`), `strokeWidth: 4.5`, `scale-125` 효과로 극대화하여 차트 상에서의 포커스 상태를 한눈에 볼 수 있도록 시각적 강조를 구현했습니다.
  - 일람표 세부 클러스터 행(`tr`)에 고유한 `id={`row-${d.id}`}` 속성을 주입하고, 선택된 활성 행(`isSelected`)에 `bg-blue-50/90 font-bold border-l-4 shadow-xs ring-1 ring-blue-100/60` 등의 세련된 피드백 효과를 강화했습니다.
- `renderAcademicProof(issue)`를 전면 정리했다.
  - 수요 강도, 정책 공백을 상단 요약 카드로 보여준다.
  - 공통 근거로 홍향희·이정화(2026), 오신휘·김혜진(2020), NABO 보고서 흐름을 연결한다.
  - 보육·돌봄 인프라 클러스터는 성낙일·박선권(2012), KICCE GIS 수요·공급 형평성 연구를 추가한다.
  - 주거·노동·생활지원 계열은 배기련 외(2021), 박미경(2022)을 우선순위 근거로 추가한다.
- `getAcademicEvidenceItems(issue)`를 추가했다.
  - 선택된 이슈 카테고리에 맞춰 논문/보고서 목록, 링크, 상세 설명, 정책 의미를 반환한다.
- `getPolicyHypothesis(issue, rawData, evidenceItems)`를 추가했다.
  - 시민제안, 국민신문고, 서울시 정책 공급, 뉴스 흐름을 합쳐 "정책이 시행된다면 시민에게 어떤 도움이 되는지"를 문장형 가설로 생성한다.
  - 보육·돌봄, 임신·난임, 주거·노동·생활지원, 정보·거버넌스, fallback 케이스를 분기 처리한다.
- 비교 분석 모달 상단에 `데이터·논문 기반 정책 가설` 영역을 추가했다.
  - 수요 건수, 정책 공백 점수, 근거 신뢰도, 논문 수를 배지로 표시한다.
  - 결론 박스에 우선 검토 액션을 강조한다.
- 비교 분석 모달 우측 하단에 `근거 논문 상세 보기` 패널을 추가했다.
  - 논문/보고서 버튼 클릭 시 선택 인덱스를 바꾸고, 제목/태그/상세/정책 의미를 표시한다.
  - 각 버튼은 외부 링크를 새 탭으로 연다.
- 모달 레이아웃을 `h-[92vh] max-h-[92vh]`, `flex-1 min-h-0`, 내부 스크롤 구조로 수정했다.
  - 상단 정책 가설 영역을 고정 높이처럼 두고, 3열 원천데이터 영역이 넘칠 때 내부 스크롤되도록 조정했다.

주의할 점:

- JSX 안에 일부 emoji 텍스트가 기존 코드 흐름대로 남아 있다. 프로젝트가 이미 해당 스타일을 쓰고 있어서 유지했다.
- 논문 URL은 외부 링크이므로 실제 제출 전 접근 가능 여부를 한 번 더 확인하면 좋다.
- `selectedEvidenceIndex`는 카드 클릭/비교 모달 열기 시 0으로 초기화된다.

### 3-2. `ReportExportModal.tsx`

핵심 목적:

- 기존의 긴 문자열 조립식 보고서 생성기를 섹션 기반 보고서 빌더로 바꿨다.
- 실무자가 포함할 섹션을 선택하면 오른쪽 미리보기와 다운로드 본문에 같은 내용이 반영되도록 했다.
- CSV 내보내기도 선택 범위의 원자료 형태로 정리했다.

주요 구현:

- 타입 추가:
  - `ReportType = 'detailed' | 'executive'`
  - `ExportFormat = 'hwp' | 'pdf' | 'excel'`
  - `SectionKey = 'background' | 'stats' | 'gaps' | 'ai' | 'academic' | 'logs'`
  - `ReportSection`
- `SECTION_LABELS` 추가로 보고서 섹션명을 중앙 관리한다.
- `getDepartmentNames(proposal)` 추가:
  - `proposal.department`와 `proposal.department_rankings[].dept_name`을 함께 보고 부서 필터링한다.
- `createCsv(proposals, selectedDept)` 추가:
  - 제안ID, 제목, 카테고리, 담당부서, 공감수, 등록일, 답변여부를 CSV로 만든다.
- 기존 `deptProposals`, `deptCategories`, 여러 `includeX` boolean을 정리하고 `enabledSections`로 통합했다.
- `scopedProposals`를 통해 부서/카테고리/자치구 필터를 한 번에 적용한다.
- `reportData`에서 총 공감수, 미답변, 답변률, 고공감 미답변, 부정 신호, 상위 카테고리, 최상위 공감 제안, gapScore 등을 계산한다.
- `sections` 배열에서 보고서 문단을 구조화한다.
- 좌측 UI:
  - 보고서 유형 선택
  - 파일 형식 선택
  - 6개 추천 섹션 체크박스
- 우측 UI:
  - Excel 선택 시 CSV 미리보기
  - HWP/PDF 선택 시 섹션 카드형 문서 미리보기
- 다운로드 버튼은 선택한 형식에 맞게 텍스트/CSV 파일을 생성한다.

주의할 점:

- 실제 `.hwp`나 `.pdf` 바이너리 생성은 아니다. 기존 흐름처럼 텍스트 기반 다운로드다.
- PDF 선택 시 파일명/미리보기 라벨은 PDF 요약 형태지만 실제 다운로드 구현이 텍스트 파일인지 확인이 필요하다. 사용자가 진짜 PDF 생성을 원하면 별도 라이브러리나 서버 처리가 필요하다.
- CSV는 UTF-8 BOM 처리가 필요한지 실제 Excel 열기 환경에서 확인하면 좋다.

## 4. 정책 분류 스크립트 변경 내용

### 4-1. `docs/정책/03_classify.py`

핵심 목적:

- 병합 충돌 흔적을 제거하고 분류 로직을 `classify_text` 중심으로 정리했다.
- 카테고리, 담당부서, 수요 점수, 부정 신호를 한 번에 반환하도록 했다.

주요 구현:

- `clean_text(text: str) -> str`
  - 비문자 입력은 빈 문자열로 처리한다.
  - 특수문자는 공백으로 정리한다.
- `score_keywords(text, keyword_dict)`
  - 라벨별 키워드 출현 횟수를 계산한다.
- `ranked_labels(scores)`
  - 점수 내림차순, 라벨명 오름차순으로 정렬한다.
- `classify_text(text)`
  - `category`
  - `categories`
  - `department`
  - `departments`
  - `demand_score`
  - `negative_signal`
  를 반환한다.
- `classify_row(row)`
  - `TITLE/title`, `content_full/content/CONTENT`를 유연하게 읽어 pandas Series로 반환한다.
- `__main__` 샘플 테스트 문구를 정리했다.

주의할 점:

- 기존 `classify_and_score(text)` 함수는 제거됐다. 다른 스크립트가 이 함수를 import하고 있다면 호출부 수정이 필요하다.
- `negative_keywords.py`의 `has_negative_signal`은 그대로 사용한다.
- `demand_score`는 현재 카테고리 키워드 적중량 * 10, 최대 100의 단순 보조 점수다.
