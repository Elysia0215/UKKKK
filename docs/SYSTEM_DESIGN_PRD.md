# 🏛️ 서울시 출산·양육 정책 의사결정 지원 대시보드 (PRD & 시스템 기획서)

> **최종 한 줄 정의**  
> 본 서비스는 **상상대로 시민 제안**, **국민신문고 민원**, **몽땅정보통 정책사업**, **뉴스 데이터**를 하나의 출산·양육 정책 분류체계로 통합하여 **시민 수요와 기존 정책 공급의 간극(Gap)을 분석**하고, **담당부서별 우선 검토 이슈와 추천 액션을 제공하는 실무자용 의사결정 지원 대시보드**입니다.

---

## 1. 프로젝트 배경 및 문제 정의 (Problem Statement)

### 🚨 현장의 문제점
* **데이터 파편화**: 공무원 실무자가 정책을 수립하거나 개선할 때, 시민들의 정책 제안(상상대로), 현장 민원(국민신문고), 기존 서울시 사업(몽땅정보통), 언론 보도(뉴스)가 각각 분산되어 있어 종합적인 상황 판단이 어렵습니다.
* **수요-공급 불일치**: 시민들의 고충이나 불만이 계속 제기됨에도 불고하고, 이것이 "신규 정책이 없어서"인지, "기존 정책의 홍보/신청 절차 문제"인지 명확히 파악하기 힘듭니다.
* **업무 피로도**: 수천 건의 파편화된 제안/민원 낱개 데이터를 일일이 열어보는 것은 과도한 행정력 낭비를 초래합니다.

---

## 2. 해결 방안: 수요-공급-이슈 갭(Gap) 분석 시스템

본 시스템은 4가지 데이터 원천을 하나의 통합 대·중·소분류 체계로 정규화한 후, **동일 주제별 [통합 이슈(Integrated Issue Card)] 단위로 묶어서 제공**합니다.

```
[ 1. 상상대로 시민제안 ] ➔ 시민이 바라는 "아이디어 / 정책 요구" (수요)
[ 2. 국민신문고 민원 ]   ➔ 시민이 겪는 "현장 고충 / 불편 신호" (시급성)
[ 3. 몽땅정보통 정책 ]   ➔ 공공이 제공 중인 "323개 현행 혜택" (공급)
[ 4. 뉴스 데이터 ]       ➔ 외부 언론의 "사회적 이슈 / 위험 신호" (확산)
           ↓
 ── [통합 정책 분류 체계 (8대 대분류)] ──
           ↓
 ── [담당 부서 1·2·3순위 매칭 & 5대 추천 액션 표출] ──
```

---

## 3. 4대 데이터 원천별 역할 정의

| 데이터 소스 | 주요 역할 | 대시보드 활용 방식 |
|---|---|---|
| **상상대로 시민 제안** | 시민 아이디어 및 정책 개선 요구 | 제안 수, 공감수, 댓글수, 대표 제안문 (수요강도) |
| **국민신문고 민원** | 현장 고충, 불만, 시급성 높은 불편 신호 | 민원 수, 반복키워드, 시급성 신호 |
| **몽땅정보통 정책** | 기존 서울시 정책 공급 데이터 (323건) | 관련 정책 수, 신청 URL, 커버리지 (공백 판단) |
| **뉴스 크롤링** | 외부 사회적 이슈 및 언론 확산 신호 | 최근 보도량, 주요 헤드라인, 이슈 확산도 |
| **담당 부서/업무분장** | 정책 수립 및 개선 실행 주체 (18개 실무팀) | 1·2·3순위 부서 후보, 전화번호, 업무 근거 |

## 4. 실무 공무원 4대 주요 업무 시나리오

1. **신규 정책 검토 모드**
   * 시민 제안과 현장 민원은 많지만 관련 기존 정책(몽땅정보통) 매칭이 낮은 경우
   * ➔ **`정책 공백 가능성 탐지`** ➔ **`[추천 액션: 신규 정책 검토]`**
2. **기존 정책 개선·홍보 모드**
   * 민원과 불만은 지속되지만 관련 기존 정책이 이미 존재하는 경우
   * ➔ **`접근성/홍보 문제 가능성`** ➔ **`[추천 액션: 기존 정책 홍보 강화 또는 신청·이용 기준 개선]`**
3. **언론 확산 모니터링 모드**
   * 뉴스 언급량과 시민 제안/민원이 동시에 급증하는 경우
   * ➔ **`사회 이슈화 가능성`** ➔ **`[추천 액션: 현장 점검·민원 대응 또는 선제 브리핑]`**
4. **부서별 맞춤 업무 지원 모드**
   * 담당부서(예: `저출생사업1팀`, `가족건강팀`)별로 관련 제안, 민원, 정책, 뉴스를 하나의 카드 단위로 묶어 자동 1차 라우팅 제공.

## 5. 핵심 판단 로직 & 5대 추천 액션 라벨링

### 🎯 5대 표준 추천 액션 라벨
1. **`신규 정책 검토`**
2. **`기존 정책 홍보 강화`**
3. **`신청·이용 기준 개선`**
4. **`부서 협업 검토`**
5. **`현장 점검·민원 대응`**

### 📊 판단 로직 매트릭스
| 데이터 조건 조합 | 시가지 판단 | 최종 추천 액션 라벨 |
|---|---|---|
| 제안 많음 + 민원 많음 + 기존 정책 적음 | 정책 공백 가능성 | **신규 정책 검토** |
| 민원 많음 + 기존 정책 있음 | 정책 체감/접근성 문제 가능성 | **기존 정책 홍보 강화** 또는 **신청·이용 기준 개선** |
| 뉴스 많음 + 민원 증가 | 사회 이슈화 가능성 | **현장 점검·민원 대응** |
| 제안 많음 + 민원 적음 | 아이디어성 수요 | **중장기 정책 검토** (지속 모니터링) |
| 여러 대분류/여러 부서에 걸침 | 복합 이슈 | **부서 협업 검토** |
| 안전·건강·위기 키워드가 많은 이슈 | 시급성 높은 이슈 | **현장 점검·민원 대응** |

## 6. MVP 우선순위 점수 산식 (Scoring Model)

$$ \text{우선순위 점수} = (\text{수요강도} \times 0.4) + (\text{시급성} \times 0.3) + (\text{정책공백} \times 0.2) + (\text{뉴스확산} \times 0.1) $$

* **수요강도 (40%)**: 상상대로 제안 수 + 공감수 + 댓글수
* **시급성 (30%)**: 국민신문고 민원 수 + 안전/건강/위기 키워드 포함 여부
* **정책공백 (20%)**: 관련 몽땅정보통 정책 매칭 수가 적을수록 높게 가산
* **뉴스확산 (10%)**: 최근 30일 관련 언론 보도량 및 이슈성 키워드

> 💡 **안전한 표현 가이드 (Q&A 대응)**  
> *"가중치는 확정 고정값이 아닌 MVP 기준 예시이며, 실제 실무자 피드백과 정책 가치관에 따라 수식을 유연하게 조정할 수 있는 구조입니다."*

## 7. 대시보드 UI/UX 화면 구성 가이드

### 1) 메인 화면: 5대 정책 이슈 퀵 카드 (Quick Board)
1. **시민 수요 증가 이슈**
2. **기존 정책 개선·홍보 필요 이슈**
3. **정책 공백 검토 이슈**
4. **언론 확산 모니터링 이슈**
5. **부서 협업 검토 이슈**

* **카드 표시 항목**: 이슈명, 대/중/소분류, 수요/민원/정책/뉴스 수량, 우선순위 점수, 담당부서 1순위 후보, 추천 액션 라벨

### 2) 상세 화면: Integrated Policy Card (4대 탭 & 고정 패널)
* **탭 1 [시민 제안]**: 상상대로 대표 제안문, 공감/댓글수, 핵심 키워드
* **탭 2 [현장 민원]**: 국민신문고 민원 내용, 반복 고충 키워드, 시급성 신호
* **탭 3 [기존 정책]**: 몽땅정보통 매칭 사업, 지원 대상, 신청 URL, 정책 커버리지
* **탭 4 [뉴스/이슈]**: 관련 뉴스 제목, 보도 추이, 외부 여론 요약
* **오른쪽 고정 패널**: **추천 액션 라벨**, **정책 공백 가능성**, **담당부서 1·2·3순위 라우팅**, **매칭 근거**, **`[1-Page 행정 보고서 자동 생성]`** 버튼

## 9. 발표(Presentation) 5단계 추천 스토리라인

1. **[문제 정의]**: *"분산된 제안·민원·정책·뉴스 데이터로 인해 공무원이 현장 수요와 공급 간극을 파악하기 어렵습니다."*
2. **[해결 방향]**: *"4대 데이터를 8개 대분류 체계로 통합하여 수요-공급 갭을 분석하는 실무자용 의사결정 지원 대시보드를 구축했습니다."*
3. **[데이터 융합]**: *"상상대로(수요) + 국민신문고(민원/시급성) + 몽땅정보통(공급) + 뉴스(여론) + 18개 실무팀 연동"*
4. **[실무 시나리오 & UI]**: *"5대 퀵 카드 ➔ Integrated Policy Card 4대 탭 ➔ 5대 추천 액션 자동 제안"*
5. **[기여 효과]**: *"불필요한 신규 사업 억제, 기존 정책 이용률 제고, 사각지대 선제 대응으로 행정 효율성 극대화"*
