# 🏛️ 서울시 출산·양육 정책 의사결정 지원 대시보드 (PRD & 시스템 기획서)

> **서비스 한 문장 정의**  
> 본 서비스는 **상상대로 시민 제안(824건)**, **국민신문고 민원(582건)**, **몽땅정보통 정책사업(323건)**, **뉴스/통계 데이터**를 하나의 출산·양육 정책 분류체계로 통합하여 **시민 수요와 기존 정책 공급의 간극(Gap)을 분석**하고, **담당부서별 우선 검토 이슈와 5대 추천 액션을 제공하는 실무자용 의사결정 지원 대시보드**입니다.

## 1. 프로젝트 배경 및 문제 정의 (Problem Statement)

### 🚨 현장의 문제점
* **데이터 파편화**: 공무원 실무자가 정책을 수립하거나 개선할 때, 시민들의 정책 제안(상상대로), 현장 민원(국민신문고), 기존 서울시 사업(몽땅정보통)이 각각 분산되어 있어 종합적인 상황 판단이 어렵습니다.
* **수요-공급 불일치**: 시민들의 고충이나 불만이 계속 제기됨에도 불구하고, 이것이 "신규 정책이 없어서"인지, "기존 정책의 홍보/신청 절차 문제"인지 명확히 파악하기 힘듭니다.
* **행정 소모도**: 수천 건의 파편화된 제안/민원 낱개 데이터를 일일이 열어보는 것은 과도한 행정력 낭비를 초래합니다.

## 2. 해결 방안: Multi-API 융합 수요-공급-민원 갭(Gap) 분석 시스템

본 시스템은 4가지 데이터 원천을 하나의 통합 8대 대분류 체계로 정규화한 후, **동일 주제별 [통합 이슈(Integrated Issue Card)] 단위로 묶어서 제공**합니다.

```
[ 1. 상상대로 시민제안 ] ➔ 시민이 바라는 "아이디어 / 정책 요구" (수요 824건)
[ 2. 국민신문고 민원 ]   ➔ 시민이 겪는 "현장 고충 / 불편 신호" (시급성 582건)
[ 3. 몽땅정보통 정책 ]   ➔ 공공이 제공 중인 "323개 현행 혜택" (공급 323건)
[ 4. 자치구 공공 통계 ] ➔ 서울시 25개 자치구 출생아수 & 보육시설 정원 (인프라 갭)
           ↓
 ── [통합 정책 분류 체계 (8대 대분류)] ──
           ↓
 ── [담당 부서 1·2·3순위 매칭 & 1클릭 일괄 행정 답변 공문 표출] ──
```

## 4. 메인 대시보드: 5대 퀵 카드 (Quick Board) 및 TOP 3 인사이트

1. **시민 수요 증가 이슈**: 공감수 및 제안 제출 건수가 급증하는 분야
2. **기존 정책 개선·홍보 필요 이슈**: 기존 사업이 존재하나 관련 불만 민원이 높게 발생하는 분야
3. **정책 공백 검토 이슈**: 시민 요구는 높으나 대응되는 몽땅정보통 정책이 적은 분야
4. **언론 확산 모니터링 이슈**: 사회적 보도 및 뉴스 언급량이 급증하는 분야
5. **부서 협업 검토 이슈**: 여성가족실, 시민건강국 등 다수 부서에 복합 연관된 분야

* **신규 TOP 3 인사이트 카드 탑재**:
  * 📦 **최다 제안 분야 TOP 3**: 1위~3위 정책 분야 목록 + 미니 프로그래스 바(게이지) + 비중(%) 표시 (클릭 시 해당 분야 데이터 필터링)
  * 📦 **최고 공감 제안 TOP 3**: 1위~3위 최다 공감 표 수(🔥 N표) + 제안 제목 목록 (클릭 시 제안 원문 팝업 모달)
* **HoverScrollText 전광판 스크롤**:
  * TOP 3 및 목록의 텍스트가 컨테이너 너비를 초과하면 마우스 호버 시 3초 정지 후 가로 스크롤(Marquee)되는 사용자 친화적 전광판 컴포넌트 탑재.

## 6. MVP 우선순위 점수 산식 (Scoring Model)

$$ \text{우선순위 점수} = (\text{수요강도} \times 0.4) + (\text{시급성} \times 0.3) + (\text{정책공백} \times 0.2) + (\text{뉴스확산} \times 0.1) $$

* **수요강도 (40%)**: 상상대로 제안 수 + 공감수 + 댓글수
* **시급성 (30%)**: 국민신문고 민원 수 + 안전/건강/위기 키워드 포함 여부
* **정책공백 (20%)**: 관련 몽땅정보통 정책 매칭 수가 적을수록 높게 가산
* **뉴스확산 (10%)**: 최근 보도량 및 이슈성 키워드

> 💡 **안전한 행정 표현 가이드**  
> * `"자동 확정/100% 판정"` 대신 ➔ **`"정책 공백 가능성 탐지"`, `"우선 검토 이슈 추천"`, `"설명 가능한 규칙 기반 매칭"`** 표현 사용  
> * 가중치(40-30-20-10)는 MVP 수식 예시이며 실무 피드백에 따라 유연하게 변경 가능한 구조입니다.

## 9. 컴포넌트별 상세 기능 명세 (2026-07-24 신규)

### 9-1. PriorityDetails (긴급 민원 처리)

- **우수제안 뱃지**: `vote_score >= 150`인 제안에 ⭐ **우수제안** 뱃지를 자동 표시합니다.
  - 개별뷰(Individual View)와 그룹뷰(Group View) 모두에서 뱃지가 표출됩니다.
  - 뱃지는 제안 카드 상단 우측에 고정 배치됩니다.
- **다중선택 OFF 버튼 UX 개선**:
  - 다중선택 모드가 비활성(OFF) 상태일 때 버튼에 `animate-pulse` 애니메이션을 적용하여 시각적 주의를 유도합니다.
  - 호버 시 "다중선택을 켜서 일괄 처리하세요" 툴팁 힌트를 표시합니다.

### 9-2. GapMatrixDashboard (정책 갭 진단)

- **공문 형식 템플릿 4종 선택 버튼**: 승인 패널(Approval Panel)에 공문 템플릿 선택 UI를 추가합니다.
  - ① **정책 보완 안내형**: 기존 정책의 미비점을 보완하는 안내 공문
  - ② **신규 사업 검토형**: 신규 정책 사업 검토를 요청하는 공문
  - ③ **기존 정책 연결형**: 기존 유사 정책과의 연결·통합을 제안하는 공문
  - ④ **조례 개정 검토형**: 관련 조례 개정 검토를 요청하는 공문
  - 선택한 템플릿에 따라 공문 본문이 자동 생성됩니다.
- **신뢰도 산정 방법론 강화**:
  - **MCDA (Multi-Criteria Decision Analysis)** 기반 가중 점수 산정
  - **데이터 삼각검증(Data Triangulation)**: Denzin(2012) 논문 근거를 기반으로, 3개 이상 데이터 소스(제안/민원/정책)의 교차 확인을 통해 신뢰도를 산출합니다.
  - 신뢰도 등급: 상(3개 소스 일치) / 중(2개 소스 일치) / 하(단일 소스)

### 9-3. MissingDataSimulator (결측치 복원 시뮬레이터)

- **담당자 피드백 시스템 (Human-in-the-loop 데이터 파이프라인)**:
  - 복원 카드(Restored Card)에 🚩 **피드백 버튼**을 추가합니다.
  - 피드백 버튼 클릭 시 오류 유형 3종 선택 모달이 표출됩니다:
    - ① **지역 오매칭**: 복원된 자치구/지역 정보가 실제와 불일치
    - ② **분류 오매칭**: 복원된 정책 대분류/소분류가 부적절
    - ③ **기타**: 자유 입력 방식의 기타 오류 신고
  - 각 피드백에 **맥락 메모**(자유 텍스트)를 추가로 입력할 수 있습니다.
  - 피드백 로그는 **localStorage**에 축적 저장됩니다.
