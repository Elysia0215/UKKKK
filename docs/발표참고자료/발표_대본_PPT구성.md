### Slide 2 & 3: 비즈니스 분석 및 문제 정의 (문제 정의 및 현황 진단)
* **시각 자료**: 기존 컴퓨존의 복잡한 부품 목록 화면 ➡️ 사용자가 겪는 장벽(Pain Point 및 VOC) ➡️ 비즈니스 가치 인포그래픽.
* **내용**: 
  - **문제 정의**: 기능은 충분히 많으나 사용자의 목적과 긴밀히 연결되어 있지 않음 (메뉴 탐색 장벽).
  - **커뮤니티 실제 VOC (퀘이사존, 다나와 등)**:
    - *"유튜브와 커뮤니티마다 추천이 다 달라 부품 검증 불가 (정보 비대칭)"*
    - *"고가 그래픽카드 구매 후 CPU 병목으로 프레임 급감 (호환성 지식 부재)"*
    - *"자가 분리 조립을 못 하는데 본체를 뜯어 유통사로 보내라는 응대 (A/S 신뢰 불안)"*
  - **컴퓨존 비즈니스 목표**: 초보 고객 이탈 방지 및 단순 반복 상담 감축.
  - **해결책 (4대 핵심 기능 설계)**: 
    1) 스마트 맞춤 PC 견적 추천
    2) 정품 무상 A/S 보증기간 조회
    3) 전문 기술 상담사 실시간 연결
    4) 컴퓨존 빠른 견적 메뉴찾기
  - *(※ 구체적인 6대 메뉴별 Pain Point 분석과 해결책 도출 과정은 발표 대본 최하단의 **[백업 자료 2]**에 수록되어 있습니다.)*
* **시간**: `45초`

> **[스크립트]**
> 컴퓨존은 연간 20만 대 이상의 PC 조립 실적을 달성한 대한민국 1등 컴퓨터 부품 쇼핑몰입니다. 저희는 기존의 6대 메뉴를 정밀 진단하였고, 커뮤니티 데이터 분석 결과 초보 고객들이 부품 호환 병목이나 A/S 접수 장벽 등 현실적인 Pain Point로 인해 구매 직전에 대거 이탈하고 있음을 확인했습니다.
> 저희는 이 병목을 해결하기 위해 4대 핵심 기능을 설계했습니다. 목돈이 지출되어 조립 실수와 사후 관리를 두려워하는 조립 PC 구매 과정을 패키지 여행 상품처럼 안전하게 리드하여 고객 이탈을 혁신적으로 방어합니다.
> 
> **(➡ Slide 3 화면 전환: AS-IS 복잡한 화면 캡처 노출)**
> 화면을 보시면 기존 홈페이지의 복잡한 레이아웃으로 인해 사용자가 탐색 시 겪는 정보 과부하를 직관적으로 확인할 수 있습니다. 이에 관한 구체적인 6대 메뉴별 상세 진단 과정은 발표 자료 마지막 페이지의 **[백업 자료 2]**에 자세히 정리해 두었습니다.

### Slide 4: 서비스 설계 ①: IA, PRD 및 와이어프레임
* **시각 자료**: 새로운 AI 피처가 포함된 인포메이션 아키텍처(IA) 구조도 (★ 프로젝트_분석_보고서.md 4절의 IA 구조를 도형화하여 첨부) ➡️ **[와이어프레임 영역 (피그마로 설계된 와이어프레임 + 메인 상품 클릭 시 코미 질문 연결 플로우)]** ➡️ PRD 요구사항 정의 리스트.
* **내용**:
  - **IA (제품 설계 구조)**: 새로운 AI 피처(견적, 보증조회, 실시간 상담, 빠른 메뉴 매핑, 상품 클릭 즉시 질의)가 포함된 흐름 정의.
  - **PRD**: 스마트 추천 예산 세이프가드, 실시간 보증 D-Day 렌더링, 외부 이탈 방지 RAG FAQ 수록.
  - **와이어프레임 (회의/피칭용)**: 대화형 모달 창 배치 및 쇼핑몰 메인의 **[부품 클릭 시 코미에게 즉시 시맨틱 스펙 질의]** 단축 여정 설계도 수록.
* **시간**: `45초`

> **[스크립트]**
> 다음으로 기획 및 요구사항을 도출하여 실체화한 **인포메이션 아키텍처(IA)와 PRD 설계** 단계입니다.
> 저희는 단순 일회성 문답 챗봇을 넘어, 컴퓨존의 방대하고 깊은 메뉴 장벽을 허물 수 있도록 4대 AI 피처를 유기적으로 연결한 6가지 갈래의 멀티 인텐트 라우팅 IA를 설계했습니다. 화면 중앙의 **[피그마로 설계된 와이어프레임 영역]**에는 쇼핑몰 메인 화면의 각 부품 카드에 장착된 '코미에게 질문하기' 버튼을 눌렀을 때, 사용자가 타이핑할 필요 없이 즉시 챗봇이 활성화되며 상세 장단점을 파악해 주는 단축 인터랙션 흐름도 회의 설계안에 함께 반영되어 있습니다. 요구사항 정의서인 PRD에는 크롤링 실데이터 호환성 검증, 제조사/컴퓨존 A/S 분기 로직 등을 수록하여 개발의 명확한 스펙 기준을 수립했습니다.

### Slide 5: 서비스 설계 ②: AI 아키텍처 및 프롬프트 전략
* **시각 자료**: 5단계 AI 추천 파이프라인 흐름도(RAG 임베딩 예시 포함) 및 2중 캐싱(1차 의도 캐시 & 2차 스펙 Aspects 캐시) 레이아웃 다이어그램.
* **내용**:
  - **AI 아키텍처 (LLM 파이프라인)**: 1) 유형 분류 ➡️ 2) 기준 도출 ➡️ 3) 예산 배분 ➡️ 4) 규칙 호환성 검사(R-01~R-05) + **ChromaDB RAG 임베딩 매칭** ➡️ 5) Gemini 리포트 생성.
  - **2중 캐싱 최적화**: 1차 의도(Intent) 분류 캐시 및 2차 상품스펙 Aspects 캐시 적용으로 호출 비용 80% 절감.
  - **Fallback 전략**: Gemini API 장애 시 로컬 DB 규칙 매칭으로 즉시 자동 전환하여 중단 없는 추천 보장.
* **시간**: `55초`

> **[스크립트]**
> 저희 추천 엔진의 **AI 아키텍처와 프롬프트 전략**을 설명해 드리겠습니다.
> 비용과 속도 최적화를 위해 규칙 연산과 호환성 검사(R-01~R-05)를 백엔드에서 자체 처리하고 자연어 리포트 생성에만 Gemini API를 활용하는 **5단계 파이프라인**을 설계했습니다. 특히 이 과정에서 비개발자 유저의 자연어 입력과 데이터베이스의 하드웨어를 연결하기 위해 **ChromaDB RAG 임베딩 의미 매칭 기술**을 이식했습니다. 
> 
> 또한, 불필요한 LLM 호출을 막는 **1차 의도 분류 캐시**와 중복 상품 질의에 즉시 대답하는 **2차 상품스펙 Aspects 캐시**의 2중 캐싱 기법을 적용해 API 호출 비용을 80% 이상 획기적으로 낮추었습니다.
> 만약 LLM API 장애가 발생하더라도, 로컬 규칙 기반 알고리즘으로 자동 폴백(Fallback)되어 에러 없이 3종 추천 결과서를 로드해주는 고가용성을 내장했습니다.
