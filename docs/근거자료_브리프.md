# UKKKK 근거 자료 브리프

> **핵심 프레이밍**: UKKKK는 서울시가 이미 투자한 자산(상상대로서울)의 **행정적 ROI를 극대화하는 생산성 촉진제**다.

---

## 1. 상상대로서울 = 서울시의 공식 투자 자산

**팩트**

- 서울시 홍보기획관 콘텐츠담당관이 운영하는 공식 시민참여 플랫폼 (idea.seoul.go.kr)
- 서울 열린데이터광장에 공공데이터로 등록 (2015.03.30 공개, 2026.07.17 최종 갱신)
- 월간 보고서 발행 중 (2026년 5월 기준: 활성 사용자 7,259명, 조회수 110,015, 제안 144건)
- 인스타그램 등 SNS 홍보 운영, 연간 우수제안 시민투표 시행
- 실제 정책 채택 사례 존재 (우수제안 → 서울시 정책 반영)

**의미**: 서울시가 예산·인력·홍보를 투입하고 있는 자랑스러운 사업. 하지만 투입 대비 **데이터 활용도**는 낮다.

## 2. 근거 ①: 정책 공백 — 제안 수 vs 정책 수 불균형

**데이터** (프로젝트 코드베이스 실측)

| 구분 | 건수 |
|------|------|
| 출산·양육 시민제안 (proposals.json) | **824건** (상상대로 전체 중 출산·양육 키워드 필터링 결과) |
| 몽땅정보통 등록 정책 (classified_policy.json) | **322건** (출산·양육 관련 정책만 추출) |

- ⚠️ 단순 총량 비교(824 vs 322)는 부적절 — 제안 간 중복·유사 내용 존재, 양쪽 카테고리 체계도 다름
- 핵심은 **카테고리별 수급 불균형**:
  - **임신·난임**: 제안 274건 (33.3%) — 가장 많은 수요이나 부서 답변율 11.3%
  - **보육·돌봄**: 제안 175건 (21.2%) — 답변율 8.0%
  - **정보·상담**: 제안 144건 (17.5%) — 답변율 7.6%
  - **일·가정양립, 주거·교통, 취약가족**: 제안은 있으나 부서 답변율 **0%** — 정책 대응 자체가 부재

**의미**: 상상대로에 출산·양육 제안이 꾸준히 올라오지만, 일부 카테고리는 답변율 0%로 사실상 방치 상태다. 이는 해당 분야의 정책 공급이 수요를 따라가지 못하고 있다는 신호이며, UKKKK는 이 **카테고리별 수급 갭을 수치로 시각화**하여 정책 기획의 우선순위를 제시한다.

## 3. 근거 ②: 구(區) 미상 결측치 97.6%

**데이터**

| 구분 | 건수 | 비율 |
|------|------|------|
| 구 정보 "미상" | 804건 | **97.6%** |
| 구 정보 있음 | 20건 | 2.4% |

**의미**: 824건 중 804건의 지역 정보가 없다. 애초에 시민 제안은 특정 구가 아니라 서울시 전체를 대상으로 쓰는 경우가 많고, 본문에 구 이름이 없는 글도 많다. 현재 상상대로에서는 이 결측이 **눈에 보이지 않는다** — 월간 보고서에도 별도 통계로 보고되지 않는다.

**UKKKK가 해결하는 방식 — "미상을 보이게 만든다"**:

- 구별 차트에서 **미상 항목을 명시적으로 시각화** → 담당 직원이 "이 제안들이 구별 통계에 반영이 안 되고 있구나"를 즉각 인식
- 인식 후 실무자가 취할 수 있는 조치:
  - 미상 제안에 수동으로 관할 구를 보정·추가
  - 서울시 전체 대상 제안은 별도 카테고리로 분리 관리
  - **결측치 자체를 데이터 품질 이슈로 인식**하고 개선 의제로 올림
- **상상대로 입력 UI 개선 기획으로 연결**: 제안 등록 시 자치구를 필수 입력값으로 추가하는 것을 제안 → 근본적 결측 방지
- 구별 보육시설 수·출생아 수·합계출산율 등 **다른 공공 통계와 연결**했을 때, 미상이 빠진 구별 차트의 불완전함이 더 명확히 드러남 → 데이터 정합성 개선의 동기 부여

**핵심 논점**: 문제는 97.6%가 미상이라는 사실 자체가 아니라, **그 사실이 지금까지 아무에게도 보이지 않았다**는 것이다. UKKKK는 결측을 가시화하고, 다른 통계와 연결해 보여줌으로써 데이터 품질 개선과 정책 의사결정을 동시에 촉진한다.

## 4. 근거 ③: 부서 답변 파편화 — 92.2% 미답변

**데이터**

| 구분 | 건수 | 비율 |
|------|------|------|
| 부서 답변 없음 (N) | 760건 | **92.2%** |
| 부서 답변 있음 (Y) | 64건 | 7.8% |

**카테고리별 답변율**

| 카테고리 | 제안 수 | 답변율 |
|---------|--------|--------|
| 일·가정양립 | 5건 | **0%** |
| 주거·교통 | 56건 | **0%** |
| 취약가족 | 12건 | **0%** |
| 다자녀 | 71건 | 2.8% |
| 출산 | 87건 | 6.9% |
| 정보·상담 | 144건 | 7.6% |
| 보육·돌봄 | 175건 | 8.0% |
| 임신·난임 | 274건 | 11.3% |

**답변 기준의 부재 — "누가 봤느냐"에 달린 구조**

| 제안 | 공감수 | 댓글수 | 답변 |
|-----|-------|-------|-----|
| 서울의료원 공공난임센터 반대 | 722 | 470 | **N** |
| 국공립 유치원 설립 요청 | 550 | 560 | **N** |
| 난임지원 횟수 철폐 | 212 | 29 | **N** |
| 어린이집 입학연기 질문 | 4 | 0 | **Y** |
| 지하철 임산부석 폐지 | 6 | 4 | **Y** |

- 공감 700+, 댓글 400+ 제안이 방치되고, 공감 4짜리가 답변을 받음
- 답변 여부에 공감수·댓글수 기반의 일관된 기준이 존재하지 않음
- 현재 구조: 부서 답변 = **담당자 개인이 우연히 확인했느냐**에 의존
- UKKKK 해결: 팀별 미답변 대시보드 + 우선순위 정렬로 **시스템 기반 대응**으로 전환

**답변 유무에 따른 시민 참여도 차이**

| 구분 | 평균 공감수 |
|------|-----------|
| 답변 있는 제안 (Y) | **233.7** |
| 답변 없는 제안 (N) | **8.7** |

- 답변 있는 제안의 공감수가 **약 27배** 높음
- 2026년 5월 보고서 기준 조회수 전월 대비 **~26% 감소** → 저조한 답변율이 참여 감소의 원인일 가능성

**의미**: 3개 카테고리는 답변율 0%로 사실상 방치 상태. 답변이 있으면 시민 참여(공감)가 27배 증가하는데, 현재 92.2%가 답변을 받지 못하고 있다.

UKKKK의 **"모아보기" + "한번에 답변" + "팀별 제안수 대시보드"** 기능이 이 문제를 직접 해결:
- 유사 제안을 클러스터링하여 부서가 **한 번의 답변으로 다수 제안에 대응** 가능
- 팀별 미답변 현황을 수치로 보여줌 → 답변 책임의 가시화
- 답변율 향상 → 시민 참여 증가 → 플랫폼 활성화의 선순환

## 5. 종합: UKKKK = 생산성 촉진제

| 상상대로 현재 상태 | UKKKK 적용 후 |
|------------------|-------------|
| 97.6% 지역 데이터 미상 (보이지 않음) | 미상을 가시화 → 인식 → 보정/필수값 추가 기획으로 연결 |
| 92.2% 부서 미답변 | 클러스터링 + 모아보기로 일괄 답변 효율화 |
| 3개 카테고리 답변 0% | 미답변 대시보드로 공백 가시화 → 대응 촉진 |
| 월간 보고서 = 단순 집계 | 정책 공백 진단 + 우선순위 + 수요-정책 매칭 |
| 카테고리별 수급 불균형 (3개 분야 답변 0%) | 카테고리별 수급 갭 시각화 → 정책 기획 우선순위 근거 |

**결론**: 서울시는 이미 상상대로에 투자하고 있다. UKKKK는 새로운 투자를 요구하는 것이 아니라, **이미 쌓인 데이터를 분석 가능한 형태로 전환**하여 행정적 의사결정에 실질적으로 활용되게 만든다. 그것이 "생산성 촉진제"의 의미다.

## 6. 문제정의 확장 논리 — "버린 게 아니라 올라간 것"

### 원래 과제 → 확장의 흐름

```
[1] 원래 문제정의
    "출산·양육 정책 수요 분석 대시보드"
        ↓ 실제 API 데이터를 꺼내봄
[2] 데이터 상태 확인
    구 미상 97.6% / 미답변 92.2% / 3개 카테고리 0% 답변
        ↓ 이 결측치가 곧 문제의 근거
[3] 확장된 문제정의
    상상대로 데이터가 행정적으로 활용 불가능한 상태 자체가 문제
        ↓ 해결책 = UKKKK
[4] UKKKK의 포지셔닝
    상상대로의 백오피스 분석 도구 (대체가 아닌 보완)
```

원래 과제(출산·양육 정책 수요 분석)는 그대로 살아있다. 실제 공공 API 데이터를 수집·분석하는 과정에서 결측치와 미답변이라는 **현실의 문제**가 드러났고, 이것이 "왜 이 대시보드가 필요한가"의 가장 강력한 근거가 된 것이다. 문제정의를 버린 것이 아니라, 데이터가 증명한 문제를 **상위 레이어로 확장**한 것이다.

### 페이지 타겟 분리

| 페이지 | 타겟 사용자 | 핵심 기능 |
|-------|-----------|----------|
| **상상대로서울** (기존) | 시민(민원인) | 제안 작성, 공감, 토론 |
| **UKKKK 관리자 페이지** (백오피스) | 공무원 실무자 | 결측치 보완 수치, 정책 공백 진단, 클러스터링 모아보기, 팀별 답변 현황, 텍스트마이닝 키워드 태그 |

UKKKK는 상상대로를 대체하지 않는다. 상상대로가 **시민의 목소리를 수집하는 프론트**라면, UKKKK는 그 목소리를 **행정이 읽을 수 있는 데이터로 전환하는 백오피스**다.

### 텍스트마이닝 → 태그 → 결측치 보완

상상대로 원본 데이터에는 구(區) 정보가 97.6% 누락되어 있지만, 제안 텍스트 안에는 지역 키워드가 포함된 경우가 있다. UKKKK는 텍스트마이닝으로 이런 키워드를 추출하고 태그화하여, 결측치를 보완한 분석 결과를 관리자 페이지에서 제공한다. 이를 통해 공무원은 구별·카테고리별 수요 현황을 파악하고, 정책 보고서 작성의 근거 자료로 활용할 수 있다.

## 7. Gemini 분석 검증 결과

Gemini가 제시한 주요 주장들을 코드베이스 실측으로 검증한 결과:

| Gemini 주장 | 검증 결과 | 비고 |
|------------|---------|------|
| "구 미상 90%" | **⚠️ 과소** — 실측 **97.6%** | 발표 시 97.6%로 정확히 언급할 것 |
| "국민신문고 원천 API 데이터 적재" | **✅ 맞음** | `civil_requests_all.json` 존재, 수집 스크립트 확인 |
| "5원 API 통합 데이터" | **✅ 맞음** | 기획서에 "5원 Multi-API 융합" 명시 (상상대로, 국민신문고, 몽땅정보통, 뉴스, 통계) |
| "AI R&R 라우팅" | **✅ 맞음** | `GapMatrixDashboard.tsx`에 카테고리→담당부서 라우팅 로직 구현 |
| "3대 가중치 시뮬레이션" | **⚠️ 부정확** — 실제로는 **3개 방법론 × 5개 가중치 팩터** | 기본/박미경(2022)/KICCE(2023) 3가지 방법론, 각각 수요·공백·시급성·실행성·신뢰도 5개 팩터 |
| "보고서(HWP/TXT) 원클릭 기안" | **✅ 맞음** | `ReportExportModal.tsx` — "기안·결재 보고서 자동생성기" 구현 |
| "데이터 피드백 루프 아키텍처" | **⚠️ 개념만** — 코드에 미구현 | 발표에서 **비전/확장성**으로 언급하는 건 OK, "구현했다"고 하면 안 됨 |

### 심사위원 방어 스크립트 수정 포인트

Gemini가 제안한 방어 스크립트는 전체적으로 좋지만, 다음을 수정해야 함:

1. **"90% 이상"** → **"97.6%"**로 정확한 수치 사용 (더 임팩트 있음)
2. **"3대 가중치"** → **"학술 논문 기반 3개 방법론의 다차원 가중치 모델"**로 표현
3. **"데이터 피드백 거버넌스 수립"** → **"데이터 피드백 루프 설계(안) 제시"**로 톤 다운 (구현은 안 됐으니까)
4. **"데이터 융합 기법을 통해 결측치를 우회"** → 실제로는 NLP 텍스트마이닝으로 제안 본문에서 지역 키워드 추출하는 방식임을 명확히
