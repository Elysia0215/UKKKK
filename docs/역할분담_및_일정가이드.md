## ⚔️ Git 충돌 Zero(0)를 위한 일일 루틴 규칙

### 1. 아침 시작 시 (Day 1~3 공통)
```bash
git checkout main
git pull origin main
git checkout feature/{본인이름}
git rebase main   # (또는 git merge main)
```

### 2. 독립된 파일만 수정
- **담당 A**: `scripts/` 디렉토리 담당
- **담당 B**: `analysis/` 디렉토리 및 `docs/데이터분석정의서.md` 담당
- **담당 C**: `정책/` 디렉토리 담당
- **담당 D**: `frontend/` 디렉토리 담당
> ⚠️ 서로의 전담 영역에 있는 코드는 직접 수정하지 않고, 수정이 필요할 경우 담당자에게 요청하여 해당 담당자가 수정 및 커밋합니다.

### 3. 퇴근 / 세션 종료 전
1. 본인 브랜치 작업 내용 커밋 및 Push:
   ```bash
   git add .
   git commit -m "feat: [담당업무] 작업 내용 요약"
   git push origin feature/{본인이름}
   ```
2. GitHub에서 `main` 브랜치 방향으로 **Pull Request(PR)** 생성
3. 팀원 확인 후 병합(Merge) 수행
