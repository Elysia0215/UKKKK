# 서울시 출산·양육 정책 수요 분석 대시보드 기획서 (최종 완성본 - 코드 동기화)

> **팀 프로젝트 최종 제출안** (작성자: 박채연 — 문제정의·사용자·KPI·Multi-API 통합 및 최신 융합 기능 반영)

---

## 1. 프로젝트 개요

본 프로젝트는 서울시(및 자치구) 출산·양육 정책 담당자를 위해 **시민 수요(상상대로 서울), 현장 민원(국민신문고), 정책 공급(몽땅정보통), 자치구 인프라 통계(출생아수·보육시설수·합계출산율)를 한 화면에서 유기적으로 연동하여 다차원 분석 및 의사결정을 지원하는 실무 백오피스 대시보드**를 구축하는 것을 목표로 한다.

> **💡 전략적 포지셔닝:** 서울시 '상상대로 서울'은 예산 공모전 등 주기적 이벤트 운영, 인스타그램 등 커뮤니티 홍보, 시민 제안의 실제 정책 실현까지 이미 막대한 행정 인력과 예산이 투입되고 있다. 그러나 **시민 제안 데이터 자체는 기준 없이 답변 여부가 결정되고, 정책 공백 진단이나 자치구별 수요 분석이 구조적으로 불가능한 상태**였다. 본 프로젝트는 이 플랫폼의 **백오피스(행정 관리 페이지)**를 새로 설계하여, 출산정책 공무원이 어떤 정책에 공백이 있는지·어떤 정책을 더해야 하는지를 데이터 기반으로 판단하고, 상상대로 플랫폼 자체를 더 활성화시키는 **의사결정 지원 에코시스템**으로 포지셔닝한다.

## 2. 문제정의

```
1. [데이터 파편화 및 수급 불균형]
행정 실무 담당자는 시민제안(상상대로), 현장 민원(국민신문고), 정책 DB(몽땅), 공공 통계(KOSIS)가 파편화되어 있어, 
시민들의 제안이 폭발하는 카테고리(수요)와 현행 서울시 정책(공급) 간의 '정책 공백(Gap)'을 정량적으로 파악하기 어렵다.

2. [부서 답변의 무작위성 및 기준 미비 실태]
실증 데이터 분석 결과, 공감 722건 + 댓글 470건이 달린 우수 제안(서울의료원 난임센터 반대)은 부서 답변이 부재한 반면, 
공감 4건짜리 단순 질문은 답변이 완료되는 등 답변 체계가 담당자 개인의 우연한 확인에 의존하고 있어 행정 신뢰도를 저해하고 있다.

3. [상상대로 결측치(97.6%) 방치]
제안 데이터의 97.6%(824건 중 804건)가 자치구 '미상(서울시 전체)'으로 작성되어 있어 지역별 수요 분석이 원천적으로 불가능한 상태였다.
```

## 3. 사용자 페르소나 및 행정 시나리오

- **대상:** 서울시 본청 여성가족실(저출생담당관, 가족담당관) 및 자치구 소속 출산·양육 정책 실무 기획자
- **주요 업무:** 시민제안 검토, 국민신문고 민원 대응, 신규 양육지원 사업 기획, 통계 비교 자료 작성
- **도입 효과:** 
  - 특정 자치구에서 반복 제기되는 쟁점 민원의 주관 부서를 **1초 만에 자동 식별(R&R 1·2·3순위 라우팅)**
  - 정책 공백 및 우선순위 점수를 통해 합리적인 예산 배분 근거 확보
  - **1클릭 일괄 답변 공문 작성 및 부서별 답변율 모니터링**으로 공문 처리 시간 70% 이상 단축 및 행정 신뢰도 대폭 향상
  - **오피스 길잡이 '새싹이'**를 통해 데이터 분석 비전문가인 공무원도 8개 탭 전체를 소속 부서 맞춤 동선으로 활용 가능 — 별도 교육 없이 백오피스 즉시 도입

## 4. AS-IS vs TO-BE

- **AS-IS:** 채널별 분절된 데이터를 엑셀 등으로 수작업 취합 및 비교하며, 부서 답변이 무작위적이고 결측 데이터(미상 97.6%)로 인해 통계 활용 불능
- **TO-BE:** 
  - **'제안 수 vs 정책 수 = 정책 공백 지수' 정량화:** 8대 정책 대분류 및 인터랙티브 갭 매트릭스를 통해 최우선 과제를 즉시 시각화
  - **[모아보기 & 1클릭 일괄 답변] 기능 구현:** 고공감 우수 제안 무응답 모아보기 필터 및 1클릭 일괄 공문 기안문 자동 추출
  - **서울시 18개 실무 팀별 답변율 대시보드:** 부서별 제안 처리 현황 및 답변율(% 수치)을 실시간 모니터링하여 행정 공백 차단
  - **5원 데이터 융합 결측치 복원:** 자치구 미상(97.6%) 데이터를 텍스트 지역 키워드 및 5원 데이터 융합으로 정제하여 공무원 실무 활용 통계로 격상
  - **한국어 텍스트마이닝 공유 유틸:** 조사/어미 제거, 유사어 통합(다둥이→다자녀 등), 동음이의어 안전장치(수유·방학·방이 등 7개 키워드)를 프로젝트 전역 공유 모듈(`textMining.ts`)로 분리 — 크롤링·실시간 수집·프론트엔드 분석에 일관 적용
  - **결측치 복원 실무자 검증 워크플로우:** 복원 결과 원문 펼치기(인라인 확인) → 체크박스 선택 확정 → 1클릭 데이터베이스 반영 → 대시보드 전체 통계 실시간 갱신
  - **키워드 CSV 내보내기:** TF-IDF 기반 키워드 목록을 대분류/중분류/소분류 포함 CSV로 다운로드하여 엑셀에서 부서별·카테고리별 필터링 즉시 활용
  - **서울시 톤 참고 답변 생성:** 상상대로 기존 답변 톤("시민 의견에 서울시가 답합니다") 및 월간 외부공유 보고서(2026년 5월 보고서) 스타일을 참고한 행정 공문체 답변 초안 생성
  - 실시간 가중치 변경 모델(기본, 박미경2022, KICCE2023) 적용 및 내부 군집 우선순위 자동 재정렬 기능 제공
  - **오피스 길잡이 '새싹이' (백오피스 어시스턴트):** 실제 공무원 백오피스 페이지에서 각 화면별 "이 화면은 왜 있는지 → 지금 할 수 있는 일 → 고급 활용 팁"을 단계별로 안내하는 인터랙티브 가이드. 상단 부서 필터 선택 시 해당 팀 맞춤 분석 워크플로우를 추천하며, 행정 용어 각주(KPI·TF-IDF·MCDA 등)를 비전문가도 이해할 수 있도록 풀어 설명
  - **ⓘ 정보 아이콘 시스템:** 모든 페이지 상단 우측에 정보(ⓘ) 아이콘을 배치하여 해당 화면의 수치 산정 기준, 데이터 출처, 방법론 설명을 클릭 한 번으로 확인 가능하게 구현. 직관적 이해와 수치 검증을 동시 지원
  - **통합 품질 관리 로그 (4종 Human-in-the-loop):** 정책 오매칭 신고(🚩)·복원 피드백(📝)·데이터 반영 이력(✅)·승인 이력(🔏) 4종을 하나의 통합 로그 뷰어에서 탭별 조회·개별 철회·일괄 초기화 가능. localStorage 기반 영구 저장으로 세션 간 이력 보존
  - **AI 답변 수정 후 승인 강제 로직:** 승인 패널에서 AI 생성 답변 초안을 담당자가 수정하면 "답변 승인" 버튼이 자동 비활성화되고 "수정 후 승인"만 활성화. AI 자동화 출력물에 대한 인간 검증(Human-in-the-loop) 거버넌스를 시스템적으로 보장

### 5-1. [중요] 비정형 시민 데이터의 한계 및 전처리 극복 방안 (미상 데이터 왜곡 방지)
- **데이터 한계 직면:** 수집된 상상대로 서울 시민제안 데이터 분석 결과, **약 90% 이상**의 제안이 특정 자치구가 지정되지 않은 **'미상(서울시 전체 대상)'** 데이터로 파악되었습니다.
- **분석 왜곡 방지 조치:** 이를 자치구별 통계 지표에 무작정 합산하여 시각화하거나 우선순위를 매길 경우, 특정 구에만 극단적인 왜곡이 발생하거나 분석의 신뢰도가 상실되는 한계가 있었습니다.
- **극복을 위한 설계 전략:**
  1. **지도 및 비교 차트 분석 축의 재구성:** 통계 분석 지도(`SeoulMap`) 및 자치구 비교 차트(`StatCharts`)의 핵심 3대 비교 분석 축을 왜곡 가능성이 높은 '시민 제안 수' 대신, 검증된 **공공 행정 지표(출생아 수, 보육시설 수, 합계출산율)** 중심으로 설계하여 신뢰성을 극대화했습니다.
  2. **미상 필터(Toggle) 탑재:** 시민 제안 목록 및 상세 차트 조회 시, '미상(서울시 전체) 데이터 포함' 여부를 실무자가 직접 선택(On/Off)할 수 있는 **25개 자치구 전용 보기 토글 필터**를 구현하여 순수 자치구 데이터와 미상 데이터를 완벽히 격리 분석할 수 있게 했습니다.
  3. **텍스트마이닝 기반 결측치 자동 복원:** 제안 원문에서 지명 키워드(노원역, 강남구, 잠실 등)를 추출하여 자치구를 자동 추정하는 규칙 기반 엔진을 구축했습니다. 특히 **동음이의어 안전장치**(수유=모유수유 vs 수유동, 방학=학교방학 vs 방학동 등 7개 키워드)를 적용하여 같은 구의 확실한 지명 키워드가 동반 출현할 때만 자치구로 인정합니다.
  4. **프로젝트 전역 텍스트 정제 유틸(`textMining.ts`):** 한국어 조사/어미 제거(`normalizeKoreanWord`), 유사어 통합(`SYNONYM_MERGES`), 불용어 사전(`STOP_WORDS`), 자치구 추정(`inferDistrict`)을 하나의 공유 모듈로 분리하여 크롤링, 실시간 API 수집, 프론트엔드 분석 전반에 일관된 텍스트 처리를 보장합니다.

## 6. 대시보드 8대 페이지(탭)별 상세 기능 요건

### [Page 1] 종합 대시보드 홈 (DashboardOverview)
- **핵심 KPI 현황판:** 서울시 전체의 출생아수 추세, 보육시설 총량, 합계출산율, 총 시민 제안 건수를 요약 시각화.
- **분야별 정책 분포:** 8대 카테고리별 시민 제안 발생 비율 및 18개 실무부서 라우팅 점유율 차트 제공.
- **실시간 민원 핫키워드 롤러:** 전광판 형태의 Marquee 배너를 통해 국민신문고 최신 유입 키워드를 스크롤 표출.

### [Page 2] 키워드 및 정책수요 분석 (CategoryDemand)
- **워드클라우드 시각화:** 비정형 시민 제안 텍스트에서 파싱된 다빈도 명사들을 직관적으로 맵핑.
- **수요 트렌드 매칭 차트:** 카테고리별 수요의 월간 추세 곡선과 실시간 네이버 LIVE 뉴스 보도량 추이를 이중축 차트로 동시 분석 및 매칭.

### [Page 3] 긴급 민원 처리 (PriorityDetails)
- **우선순위 랭킹 알고리즘 점수판:** 시민 요구 강도 + 정책 공급 공백 + 사회적 시급성 + 뉴스 보도량 가중합에 **장기 미해결 가산점**을 합산하여 1~100위 랭킹 산출.
- **⭐ 우수제안 뱃지:** 공감 150표 이상의 제안에 노란 `⭐ 우수제안` 뱃지를 자동 부착 (개별 리스트뷰·그룹 확장뷰 양쪽 적용).
- **다중선택 힌트:** 다중선택 OFF 상태에서 `animate-pulse` 깜빡임 + 호버 툴팁으로 "복수 제안 일괄 처리" 기능의 존재를 직관적으로 안내.
  
  > 💡 **우선순위 랭킹 알고리즘 가중치 설계 근거 (주석 가이드)**

### [Page 8] 결측치 복원 & 로그 (MissingDataSimulator + UnifiedLogViewer)
- **5원 데이터 융합 결측치 복원 엔진:** 구 미상(97.6%, 824건 중 804건) 제안 원문에서 지명 키워드를 추출하여 자치구를 자동 추정. 확실한 지명이 없는 건은 '서울시 전체(공통)'로 정직하게 분류.
- **동음이의어 안전장치:** '수유'(모유수유 vs 수유동), '방학'(학교방학 vs 방학동), '구의'(각 구의 vs 구의동), '방이', '보문', '가산', '신월' 7개 키워드는 같은 구의 확실한 지명 키워드가 동반 출현할 때만 자치구로 인정하는 `AMBIGUOUS_DISTRICT_KEYWORDS` 체계 적용.
- **담당자 피드백 기반 품질 개선 파이프라인 (Human-in-the-loop):** 복원 결과 카드에 🚩 피드백 버튼을 탑재하여 담당자가 오매칭(지역/분류/기타) 유형과 맥락 메모를 기록. localStorage에 로그 축적 → 페이지 하단에 피드백 로그 섹션 자동 생성. API 연동 시 `applyFeedbackKeywords()` 함수로 동음이의어 사전 자동 업데이트 가능. 데모용 목업("구의" 오매칭 예시) 1건 포함.
- **일괄 배치 복원 + 복원 불가 분류:** 전체 구 미상 건을 20건씩 비동기 배치 처리하며, 복원 성공/복원 불가(서울시 전체)로 필터링 가능.
- **실무자 검증 워크플로우:**
  - **원문 펼치기:** 각 복원 결과 카드에서 제안 원문을 인라인으로 즉시 확인 (외부 링크 이동 없이 검증 가능)
  - **원문 링크:** 상상대로 원본 제안 URL로 직접 이동하여 할루시네이션 여부 교차 확인
  - **체크박스 선택:** 전체 선택 / 개별 선택으로 복원 성공 건 중 반영할 건만 확정
  - **1클릭 데이터베이스 반영:** 선택 확정된 건을 즉시 대시보드 데이터에 반영 → 모든 탭의 통계 실시간 갱신
- **한국어 텍스트마이닝 공유 유틸(`textMining.ts`):** 조사/어미 제거(`normalizeKoreanWord`), 유사어 통합(`SYNONYM_MERGES`: 다둥이→다자녀, 임산부석→임산부배려석 등), 불용어 사전(`STOP_WORDS`), 자치구 추정(`inferDistrict`)을 프로젝트 전역 공유 모듈로 분리하여 키워드 탐색, 결측치 복원, 향후 실시간 크롤링 등에 일관 적용.
- **키워드 CSV 내보내기:** TF-IDF 기반 키워드 목록을 순위/키워드/텍스트빈도(TF)/포함제안수(DF)/TF-IDF/주연관부서/대분류/중분류/소분류 컬럼 구조의 CSV로 다운로드. BOM 포함으로 한글 엑셀 즉시 호환.
- **통합 품질 관리 로그 뷰어 (UnifiedLogViewer):**
  - 대시보드 전체에서 발생한 4종 로그를 하나의 통합 뷰어에서 탭별로 조회: ① 정책 오매칭 신고(🚩) — 다른 탭에서 "관련없음" 신고된 건, ② 복원 피드백(📝) — 결측치 복원 결과에 대한 담당자 의견, ③ 데이터 반영 이력(✅) — 복원 데이터를 대시보드에 적용한 기록, ④ 승인 이력(🔏) — 정책 갭 진단 탭에서의 답변 승인·수정후승인·반려 내역
  - 각 로그 항목에 **개별 철회(Revoke) 버튼** 탑재: 다음 개발자가 승인하기 전에 잘못된 로그를 즉시 취소 가능
  - 탭별 일괄 초기화(Clear All), 10건 이상 시 접기/펼치기, localStorage 기반 세션 간 영구 보존

### [전체] 오피스 길잡이 '새싹이' (OfficeAssistant)
- **백오피스 어시스턴트 컨셉:** 실제 민원 백오피스 페이지인 만큼 수요분석·민원처리·정책갭진단·결측치복원 등 기능이 다양할 수밖에 없다. 기능 수를 줄이면 실무 활용도가 떨어지고, 그대로 두면 직관적 이해가 어렵다. **오피스 길잡이 '새싹이'는 이 간극을 메우기 위해 설계된 백오피스 어시스턴트**로, 우측 하단 캐릭터 클릭 시 현재 활성 탭과 연동된 맞춤형 안내를 제공.
- **10개 탭별 3단계 가이드:** 각 화면마다 ① 이 화면은 왜 있는지(목적) → ② 지금 할 수 있는 일(단계별 사용법) → ③ 고급 활용 팁(실무 활용 노하우)을 체계적으로 안내.
- **팀별 맞춤 워크플로우 추천:** 상단 헤더에서 소속 부서(저출생사업1팀, 가족담당관 등)를 선택하면 새싹이가 해당 팀의 R&R에 맞는 분석 동선을 자동 추천. 예: "저출생사업1팀 → 수요현황 → 긴급민원 → 정책갭진단 순서로 검토".
- **행정 용어 각주 시스템:** KPI, TF-IDF, MCDA, 삼각검증, GeoJSON 등 비전문가가 이해하기 어려운 용어를 각 페이지 가이드 하단에 풀어 설명. 공무원 실무자가 데이터 분석 배경지식 없이도 대시보드를 활용 가능.
- **ⓘ 정보 아이콘 연동:** 모든 페이지 상단 우측의 ⓘ 아이콘 클릭 시 해당 화면의 데이터 출처·수치 산정 기준·방법론을 즉시 확인. 직관적 이해와 수치 검증을 동시 지원.

## 7. 맞춤 보고서 다운로드 기능 (`ReportExportModal` 연동)

- 우측 상단 보고서 다운로드 기능을 현재 대시보드 화면 상태(선택 자치구 및 카테고리 필터값)와 동적 연계하여 맞춤형 한글(HWP) 및 텍스트(TXT) 보고서가 자동으로 조립되어 내려받아지도록 고도화.
- **보고서 구성 6대 행정 표준 섹션 (대수수정 융합 최종안):**
  1. **추진 배경 및 목적:** 시민 요구 통합 모니터링 취지 및 실무 R&R 수립 당위성 명세.
  2. **전체 수요/현장 데이터 현황:** 선택된 자치구/카테고리가 연계 동적 적용된 시민제안 총량 및 공감수 실증 지표.
  3. **종합 Gap 진단 및 정책 가설:** 행정 만족도와 개선 액션을 도출하는 정책 공백 분석 결과 기술.
  4. **AI 기반 행정 종합의견:** 분야별/부서별 종합 행정 권고 및 이슈강도 요약.
  5. **학술 연구 및 통계 근거 대조:** 대시보드 판단 논리를 보증하는 학술적 선행 연구 및 참고 자료 대조.
  6. **행정 조치 피드백 로그:** 결재선 승인 모달에서 실무자가 확정한 액션 피드백 이력 자동 연동.
