# 서울시 출산·양육 정책 수요 분석 대시보드 기획서 (최신 융합 반영)

> 4인 팀 프로젝트 최종 통합안 (담당: 박채연 — 문제정의·사용자·KPI·Multi-API 통합)

---

## 1. 프로젝트 개요

서울시(또는 자치구) 출산·양육 정책 담당자를 위한, **지역·기간·정책 분야별 시민 수요(상상대로 서울)와 현장 민원(국민신문고), 정책 공급(몽땅정보통), 자치구 인프라 통계를 한 화면에서 교차 검토하는 실무 의사결정 지원 대시보드**를 기획한다.

---

## 2. 문제정의

```
정책 담당자는 시민제안(상상대로 서울), 현장 민원(국민신문고), 지원사업 정보(몽땅정보통), 자치구 통계가
서로 다른 채널·부서에 흩어져 있어, 특정 지역에서 어떤 정책 수요가 급증하고 있는지 빠르게 파악하고
우선순위를 정하기 어렵다.

특히 출산·양육 정책은 여성가족실·시민건강국·복지실 등 5개 이상 조직, 18개 실무 팀에
생애주기별로 파편화되어 있어, 종합적 관점의 정책 공백(Gap)을 발견하기 힘든 구조적 한계가 존재한다.
```

---

## 3. 사용자 페르소나

```
서울시 본청 여성가족실(저출생담당관, 가족담당관) 및 자치구 소속
출산·양육 정책 담당자 및 기획자

현재 하는 일: 시민제안, 민원, 지원사업 안내, 자치구 출생/보육 통계를 각각 수작업으로 비교·취합
의사결정: 어떤 자치구 및 분야(임신/출산/보육/다자녀)에 정책 예산을 우선 투입하고 1클릭 일괄 답변을 작성할 것인가
```

---

## 4. AS-IS vs TO-BE

```
AS-IS: 채널별 파편화된 데이터를 수작업 비교하며, 단순 민원 1:1 응대에 행정력 소모
TO-BE: 8대 대분류 + 자치구 지도 시각화 기반으로 유사 제안을 AI 군집화하고,
       1클릭 원스톱 행정 답변 공문 자동 생성 및 포털 원문 검증(URL 1초 확인) 제공
```
