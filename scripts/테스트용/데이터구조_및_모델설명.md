# 서울시 출산정책 제안 데이터 분류/담당매칭 데이터 구조

## 1. 입력 파일

### 제안 데이터
- 파일: `서울시_출산정책_제안데이터_426건_2026-07-21.csv`
- 단위: 시민 제안 1건 = 1행
- 주요 원본 컬럼: 제안ID, 카테고리, 제안제목, 제안본문내용, 등록일자, 공감수, 댓글수, 답변여부, 자치구, 담당부서, 원문URL, 연동 권익위 민원수

### 업무담당 데이터
- 파일: `출산정책관련 업무담당.xlsx`
- 단위: 업무담당자/담당업무 1건 = 1행
- 주요 컬럼: 담당부서, 직원명, 전화번호, 담당업무, 부서위치

## 2. 추가 생성 컬럼

| 컬럼명 | 의미 |
|---|---|
| 대분류 | 제안의 정책 영역 1차 분류 |
| 중분류 | 실제 사업/업무 단위에 가까운 2차 분류 |
| 세분류 | 제안 내용을 더 구체화한 3차 라벨 |
| 정책흐름 | 임신 전, 임신 중, 출산 직후, 영유아/초등 양육기 등 생애주기 흐름 |
| 분류근거_키워드 | 분류에 사용된 핵심 키워드 |
| 분류신뢰도 | 키워드 점수 기준 상/중/하 |
| 담당매칭방식 | 담당자 후보를 산출한 방식 |
| 1순위_담당부서 | 가장 관련성이 높은 담당부서 후보 |
| 1순위_전화번호 | 1순위 담당 전화번호 |
| 1순위_담당업무요약 | 1순위 담당자의 담당업무 일부 요약 |
| 1순위_매칭근거 | 점수 및 매칭 키워드 |
| 2순위_담당부서/전화번호/매칭근거 | 차순위 후보 |
| 3순위_담당부서/전화번호/매칭근거 | 차차순위 후보 |

## 3. 분류 체계

대분류는 다음 8개로 구성했다.

1. 임신·난임·생식건강
2. 출산·산후 초기지원
3. 보육·돌봄 인프라
4. 다자녀·양육비·생활지원
5. 주거·교통·도시생활환경
6. 일·가정 양립·부모 노동
7. 취약·다양가족 사각지대
8. 정보·상담·교육·거버넌스

## 4. 사용 모델/방법론

이번 파일 생성에는 KMeans, LLM, 딥러닝 모델을 사용하지 않았다.

사용 방식은 **규칙 기반 키워드 점수화(rule-based keyword scoring)** 이다.

### 이유
- 담당부서 매칭은 “왜 이 부서/담당자인지” 설명 가능해야 한다.
- KMeans는 비지도 군집이라 군집번호가 자동으로 생성되고, 업무담당자 1·2·3순위 라우팅 근거를 설명하기 어렵다.
- 따라서 실제 업무분장 텍스트와 제안 텍스트의 키워드 연결성을 점수화했다.

## 5. 담당자 매칭 로직

담당 매칭 점수는 다음 요소를 합산했다.

1. 제안 제목/본문의 핵심 키워드
2. 기존 카테고리/담당부서 힌트
3. 새로 만든 대분류/중분류/세분류
4. 업무담당 파일의 담당업무 텍스트
5. 수작업으로 정의한 담당업무 핵심 키워드
6. 대분류/중분류별 관련 전화번호 부스트

점수가 높은 순서대로 1순위, 2순위, 3순위 담당 후보를 산출했다.

## 6. 해석 주의사항

이 결과는 최종 행정 배정이 아니라, 대시보드/기획안용 **1차 라우팅 후보**이다. 특히 분류신뢰도가 낮거나 매칭근거 점수가 낮은 행은 사람이 최종 검토해야 한다.
