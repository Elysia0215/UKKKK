# 공공데이터 지표 탭 적용 가이드

## 제공 파일의 역할

- `table_manifest.csv`: PDF에서 발견한 표의 페이지·제목·단위·파일 위치
- `all_table_cells.csv`: 모든 표의 비어 있지 않은 셀을 감사 가능한 긴 형식으로 통합
- `all_numeric_mentions.csv`: 본문과 주석에 등장하는 숫자 및 원문 문맥
- `tables/*.csv`: 원표별 CSV. 병합 셀과 다단 표를 검증할 때 사용
- `metadata.json`: 출처, 기준연도, 공표일, 잠정치 여부 및 주의사항

## 실제 서비스에 넣기 전에 해야 할 일

1. 원표별 CSV를 지표 스키마로 정규화한다.
2. `2025p`는 화면에는 `2025년 잠정`으로 표시하고 데이터에는 `is_provisional=true`를 저장한다.
3. 명·천 명·인구 1천 명당 명·세·%·%p 등의 단위를 절대 혼합하지 않는다.
4. 전국, 시도, 시군구의 지역 레벨과 상위 지역 코드를 함께 저장한다.
5. 원자료의 반올림 수치로 합계를 다시 계산하지 않는다. PDF에 기재된 공식 합계를 우선한다.
6. 모든 지표에 PDF 페이지, 원표 제목, 기준연도, 공표일, 출처를 남긴다.
7. 2026년 8~9월 확정치 발표 후 동일 지표 키로 잠정치를 교체하되 이력은 보존한다.

## 권장 데이터 모델

```text
indicator_id       고정 영문 식별자
indicator_name     화면용 한글 지표명
category           출생 / 사망 / 자연증가
period             연도 또는 월
region_code        행정구역 코드
region_name        지역명
region_level       nationwide / sido / sigungu
dimension_name     연령, 성별, 출산순위 등 차원명
dimension_value    30-34세, 여성, 첫째아 등 차원값
value              숫자
unit               명 / 천 명 / 인구 1천 명당 명 / 세 / % / %p
is_provisional     true
source_title       2025년 인구동향조사 출생·사망통계(잠정)
source_org         국가데이터처(구 통계청)
source_page        PDF 물리 페이지
published_at       2026-02-25
```

## 화면 권장 구성

- 상단: 기준연도, 지역, 지표군, 잠정/확정 필터
- KPI: 출생아 수, 합계출산율, 사망자 수, 자연증가
- 추이: 연도별 출생·사망·자연증가 선 그래프
- 지역 비교: 시도별 출생률·사망률·자연증가율 막대 또는 지도
- 상세: 연령별 출산율, 모 평균 출산연령, 출산순위, 출생성비, 월별 출생·사망
- 모든 카드: 단위, 기준연도, 잠정치 배지, 출처·공표일, 원표 보기 제공

## 안티그래비티용 구현 프롬프트

다음 구분선 아래 내용을 프로젝트와 CSV 묶음을 함께 첨부한 뒤 그대로 입력한다.

---

당신은 시니어 데이터 엔지니어이자 프론트엔드 개발자다. 기존 프로젝트의 `공공데이터 지표` 탭에 첨부된 2025년 출생·사망통계 잠정 데이터를 추가하라. 분석만 하지 말고 구현·데이터 적재·검증까지 완료하라.

### 1. 먼저 조사할 것

프로젝트 구조, 실행 명령, 공공데이터 지표 탭의 라우트와 컴포넌트, 현재 지표 데이터 모델/API/DB/fixture, 디자인 토큰, 차트 라이브러리, 테스트 방식을 먼저 찾아라. 기존 구조와 디자인 시스템을 유지하고 전면 재작성하지 마라. 실제 데이터 경로가 있으면 fixture로 대체하지 마라.

### 2. 입력 자료

- `metadata.json`: 데이터셋 메타데이터와 잠정치 주의사항
- `table_manifest.csv`: 원표 목록
- `all_table_cells.csv`: 모든 표 셀의 감사용 통합본
- `all_numeric_mentions.csv`: 본문 숫자의 원문 문맥
- `tables/*.csv`: 원표별 추출 결과
- 원본 PDF: 수치 검증의 최종 기준

`all_numeric_mentions.csv`는 검색·근거 문맥용이지 차트 값을 직접 만드는 테이블이 아니다. 화면용 지표는 `tables/*.csv`를 원본 PDF와 대조해 정규화하여 생성하라.

### 3. 정규화 스키마

다음 필드를 기존 데이터 모델에 안전하게 매핑하라.

`indicator_id, indicator_name, category, period, region_code, region_name, region_level, dimension_name, dimension_value, value, unit, is_provisional, source_title, source_org, source_page, published_at`

- 수치는 쉼표를 제거해 number로 저장한다.
- `-`와 음수 기호를 구분하고 유니코드 마이너스도 정상 처리한다.
- `2025p`는 `period=2025`, `is_provisional=true`로 저장한다.
- 전국/시도/시군구를 구분하고, 시군구에는 상위 시도 정보를 연결한다.
- 단위가 다른 값은 동일 series에 넣지 않는다.
- 공식 합계는 하위 값을 재합산하지 말고 원표 값을 사용한다.
- 중복 적재 방지를 위해 `indicator_id + period + region_code + dimension_value + is_provisional` 조합에 고유 제약 또는 upsert를 적용한다.
- 각 레코드에 source_page와 원표 제목을 보존한다.

### 4. 최소 구현 지표

다음 지표를 우선 지원하고, 원표에서 확인되는 나머지 통계도 동일 스키마로 확장하라.

- 출생아 수, 조출생률, 합계출산율
- 모 연령별 출산율과 평균 출산연령
- 출산순위별 출생아 수와 구성비
- 결혼생활 기간별 출생
- 출생성비
- 월별·시도별·시군구별 출생
- 사망자 수, 조사망률
- 성·연령별 사망자 수와 구성비
- 사망 장소별 사망
- 월별·시도별 사망
- 자연증가와 자연증가율
- 시도별 자연증가

### 5. 화면 구성

공공데이터 지표 탭 안에 `인구동향` 또는 기존 체계에 맞는 하위 카테고리를 추가한다.

1. 필터: 기준연도, 지역 레벨, 지역, 출생/사망/자연증가, 지표, 잠정/확정
2. KPI 카드: 2025년 출생아 수, 합계출산율, 사망자 수, 자연증가. 전년 대비 증감과 증감률을 함께 표시
3. 장기 추이: 1981~2025년 출생아·사망자·자연증가. 단위가 다른 비율은 별도 차트 또는 보조축을 명확히 표시
4. 지역 비교: 2025년 시도별 출생률·사망률·자연증가율. 지표 선택과 정렬 지원
5. 상세 차트: 연령별 출산율, 출산순위, 월별 추이, 성·연령별 사망
6. 데이터 표: 현재 필터 결과를 정렬·검색하고 CSV로 내려받기
7. 출처 패널: 자료명, 기관, 공표일, 기준연도, PDF 페이지, 잠정치 주의사항

모든 카드와 차트에 단위를 표시한다. `잠정` 배지를 눈에 띄게 보여주되 오류처럼 빨간색으로 표현하지 않는다. 데이터가 없는 경우 0으로 바꾸지 말고 `데이터 없음`으로 표시한다. 색상만으로 출생·사망·자연증가를 구분하지 말고 범례·라벨을 함께 제공한다.

### 6. 데이터 검증

반드시 다음 검증을 자동화하라.

- 2025년 전국 출생아 수: 254,457명
- 2025년 전국 사망자 수: 363,389명
- 2025년 전국 자연증가: -108,931명
- 2025년 조출생률: 5.0명(인구 1천 명당)
- 2025년 조사망률: 7.1명(인구 1천 명당)
- 2025년 자연증가율: -2.1명(인구 1천 명당)
- 2025년 합계출산율: 0.80명
- `출생아 수 - 사망자 수 = 자연증가`가 명 단위 원자료에서 일치하는지 검사
- 숫자 파싱, 음수, 소수점, 단위, 잠정치, 지역 계층, 중복 upsert 테스트

PDF 주의사항에 따라 반올림된 천 명 단위 표의 하위합과 상위합 불일치는 오류로 처리하지 말고 경고로 기록한다.

### 7. 성능·접근성·회귀 방지

대량 지역 데이터는 서버 또는 메모이제이션된 selector에서 필터링하고 불필요한 전체 재렌더링을 피한다. 모바일과 데스크톱을 모두 지원한다. 키보드 탐색, 포커스 표시, 차트 대체 표, 충분한 색 대비를 제공한다. 기존 공공데이터 지표와 다른 화면/API가 깨지지 않도록 lint, typecheck, unit test, integration test, build를 실행한다.

완료 후 다음 순서로 보고하라.

1. 발견한 기존 구조와 데이터 연결 방식
2. 변경한 파일과 기능
3. 정규화한 지표 수와 레코드 수

5. 실행한 테스트와 빌드 결과

구현 도중 PDF 추출값과 원표가 다르면 추출값을 임의 보정하지 말고 원본 PDF를 기준으로 수정하고 검증 근거를 남겨라.
